# 现有系统对接 / Existing System Integration

## 概述 / Overview

系统集成是MBL数字矿山管控塔解决方案的核心能力之一，旨在打通客户现有的信息孤岛，实现业务流程的端到端自动化和数据驱动。本章节将详细阐述我们需要对接的现有系统，包括已确认必须对接的系统和我们推荐对接以实现方案价值最大化的系统。

System integration is one of the core capabilities of the MBL Digital Mine Control Tower solution, aiming to break down the customer's existing information silos and achieve end-to-end automation and data-driven business processes. This chapter details the existing systems we need to integrate with, including those confirmed as mandatory and those we recommend for maximizing the solution's value.

## 1. 已确认的系统对接 / Confirmed System Integration

根据初步调研，地磅系统是当前已确认必须对接的核心外部系统。

Based on preliminary investigation, the weighbridge system is the core external system confirmed for mandatory integration.

### 地磅系统 / Weighbridge System

| 项目 / Item | 描述 / Description |
| :--- | :--- |
| **对接系统名称 / Integration System Name** | 地磅系统 / Weighbridge System |
| **数据方向 / Data Direction** | 单向流入管控塔 / Unidirectional flow into the Control Tower |
| **核心数据项 / Core Data Items** | - 车号 / Vehicle ID<br>- 皮重 (空车重量) / Tare Weight<br>- 毛重 (满载重量) / Gross Weight<br>- 净重 (货物重量) / Net Weight<br>- 称重时间戳 / Weighing Timestamp |
| **建议集成方式 / Suggested Integration Method** | 1. **数据库中间表 (优先) / Database Middleware Table (Preferred)**: 地磅系统将其称重记录实时写入一个共享的数据库表中，我方系统从中定时读取。<br>2. **文件传输 / File Transfer**: 地磅系统定时生成称重记录文件（如CSV格式），通过SFTP/FTP传输至指定服务器，我方系统进行解析。<br>3. **API调用 / API Call**: 如果地磅系统提供API接口，我方将通过API进行调用。 |
| **业务价值 / Business Value** | - **实现装卸载重量自动采集**: 避免人工录入错误，提高数据准确性。<br>- **计算运输效率KPI**: 为计算单车运量、周期时间等核心KPI提供基础数据。<br>- **防止作弊**: 结合GPS数据，可验证车辆是否真实过磅。<br><br>- **Automate Load/Unload Weight Collection**: Avoids manual entry errors and improves data accuracy.<br>- **Calculate Transport Efficiency KPIs**: Provides essential data for calculating core KPIs like single-truck payload and cycle time.<br>- **Prevent Fraud**: Verifies if vehicles have genuinely passed the weighbridge by cross-referencing with GPS data. |

## 2. 推荐的系统对接 / Recommended System Integration

为了充分发挥管控塔的价值，我们强烈建议将以下系统也纳入集成范围。

To fully leverage the value of the Control Tower, we strongly recommend including the following systems in the integration scope.

### ERP 系统 / ERP System

| 项目 / Item | 描述 / Description |
| :--- | :--- |
| **对接系统名称 / Integration System Name** | ERP 系统 / ERP System |
| **数据方向 / Data Direction** | 双向交互 / Bidirectional Interaction |
| **核心数据项 / Core Data Items** | - **(ERP -> 管控塔)**: 运输订单、日/周生产计划、物料信息 / **(ERP -> CT)**: Transport orders, daily/weekly production plans, material information.<br>- **(管控塔 -> ERP)**: 实际运输完成量、成本核算数据 / **(CT -> ERP)**: Actual transported volume, cost accounting data. |
| **建议集成方式 / Suggested Integration Method** | API 对接 / API Integration |
| **业务价值 / Business Value** | - **业务源头数字化**: 实现从生产计划到运输执行的无缝衔接。<br>- **自动创建任务**: 根据ERP订单自动在Hauling系统中生成运输任务。<br>- **成本闭环**: 将实际发生的运输数据回传ERP，用于精确的成本核算。<br><br>- **Digitize Business Origin**: Enables seamless connection from production planning to transport execution.<br>- **Automate Task Creation**: Automatically generates transport tasks in the Hauling system based on ERP orders.<br>- **Close the Cost Loop**: Feeds actual transport data back to the ERP for accurate cost accounting. |

### 驳船物流系统 / Barge Logistics System

| 项目 / Item | 描述 / Description |
| :--- | :--- |
| **对接系统名称 / Integration System Name** | 驳船物流系统 / Barge Logistics System |
| **数据方向 / Data Direction** | 单向流入管控塔 / Unidirectional flow into the Control Tower |
| **核心数据项 / Core Data Items** | - 船期计划 / Barge Schedule<br>- 泊位信息 / Berth Information<br>- 预计到港/离港时间 (ETA/ETD) / Estimated Time of Arrival/Departure<br>- 装船量需求 / Loading Volume Requirement |
| **建议集成方式 / Suggested Integration Method** | API 对接 或 数据库中间表 / API Integration or Database Middleware Table |
| **业务价值 / Business Value** | - **实现车船高效协同**: 根据船期动态调整车队调度，减少车辆在码头的无效等待。<br>- **优化库存管理**: 结合船期和运输进度，优化码头堆场库存，避免积压或短缺。<br><br>- **Achieve Efficient Truck-Barge Coordination**: Dynamically adjust fleet dispatch based on the barge schedule to reduce idle waiting time at the port.<br>- **Optimize Inventory Management**: Optimizes port yard inventory by combining barge schedules with transport progress to prevent backlogs or shortages. |

### 采购/库存系统 / Procurement/Inventory System

| 项目 / Item | 描述 / Description |
| :--- | :--- |
| **对接系统名称 / Integration System Name** | 采购/库存系统 / Procurement/Inventory System |
| **数据方向 / Data Direction** | 双向交互 / Bidirectional Interaction |
| **核心数据项 / Core Data Items** | - **(库存 -> 管控塔)**: 备件实时库存水平 / **(Inventory -> CT)**: Real-time spare parts inventory levels.<br>- **(管控塔 -> 采购)**: 备件采购申请 / **(CT -> Procurement)**: Spare parts procurement requests. |
| **建议集成方式 / Suggested Integration Method** | API 对接 / API Integration |
| **业务价值 / Business Value** | - **智能备件管理**: 设备管理系统可根据维护计划和实时库存，自动触发采购申请。<br>- **降低库存成本**: 通过精准的需求预测和库存联动，降低安全库存水平，减少资金占用。<br><br>- **Intelligent Spare Parts Management**: The Equipment Management System can automatically trigger procurement requests based on maintenance plans and real-time inventory.<br>- **Reduce Inventory Costs**: Lowers safety stock levels and reduces capital occupation through accurate demand forecasting and inventory linkage. |
